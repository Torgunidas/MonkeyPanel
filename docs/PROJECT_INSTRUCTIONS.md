# MonkeyPanel / GorIlliaPanel — Instrukcja Projektu

## Najważniejsza zasada

Arkusz trenera jest source of truth.

Nie zastępujemy workflow trenera.

Aplikacja:
- poprawia UX,
- zapisuje dane,
- analizuje dane.

## Stack MVP

- HTML / CSS / JS
- Google Sheets
- Apps Script
- PWA
- localStorage

## Priorytety

1. UX treningu na telefonie.
2. Synchronizacja z arkuszem.
3. Diagnostyka.
4. GorIlliaPanel.

## Zakazane overengineering

Na MVP NIE:
- budujemy dużego backendu,
- robimy mikroserwisów,
- używamy Kubernetes,
- tworzymy własnej bazy.

## UX

Mobile-first.

Jedno zadanie na ekran.

Minimal friction.
